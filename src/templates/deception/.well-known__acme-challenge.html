ACME Challenge Response

This endpoint is used for domain validation via the ACME protocol (Let's Encrypt).

The server is configured to respond to HTTP-01 challenges for domain ownership verification.

Typical challenge token format:
random_string_here.another_random_string

For a valid challenge request (e.g., /.well-known/acme-challenge/<token>), the server would return the token value as plain text.

Note: This is a simulated response. Actual ACME challenge responses are plain text, not HTML. This page is presented only when the challenge path is accessed without a specific token.
