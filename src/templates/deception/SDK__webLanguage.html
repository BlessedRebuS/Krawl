Overview

The WebLanguage SDK provides a simple interface for developers to add multilingual support to web applications. It includes utilities for language detection, translation lookup, and dynamic content switching.

Installation

Include the SDK via a script tag or install via npm:

<script src="https://cdn.example.com/weblanguage-sdk/v1.0.0/weblanguage.min.js"></script>

Or using npm:

npm install @example/weblanguage-sdk

Basic Usage

Initialize the SDK with your default language and a translation map:

// Initialize SDK
WL.init({
    defaultLang: 'en',
    translations: {
        en: { greeting: 'Hello', farewell: 'Goodbye' },
        es: { greeting: 'Hola', farewell: 'Adiós' }
    }
});

// Detect user language
const lang = WL.detectLanguage() || WL.options.defaultLang;

// Get translated string
const greeting = WL.t('greeting', lang);
console.log(greeting); // Outputs based on detected language

API Reference

WL.init(options)

Initializes the SDK.

  • defaultLang (string): Language code to use when no match is found.
  • translations (object): Mapping of language codes to key/value translation objects.

WL.detectLanguage()

Returns the user's preferred language based on navigator.language or returns null if undetectable.

WL.t(key, lang)

Retrieves the translated string for key in the specified lang. Falls back to defaultLang if missing.
