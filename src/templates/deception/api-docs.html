Example Corp Public API

Version 1.2.0 – Last updated: 2024-10-12

Authentication

All requests must include an Authorization header with a Bearer token obtained from the /auth/token endpoint.

Authorization: Bearer <your_access_token>

GET /users

Retrieve a paginated list of users.

Query Parameters

  • page (integer, optional) – Page number, default 1.
  • limit (integer, optional) – Items per page, default 20, max 100.
  • search (string, optional) – Filter by username or email.

Response (200 OK)

{
  "page": 1,
  "limit": 20,
  "total": 152,
  "data": [
    {"id": 101, "username": "alice", "email": "alice@example.com", "created_at": "2023-05-14T08:30:00Z"},
    {"id": 102, "username": "bob",   "email": "bob@example.com",   "created_at": "2023-06-02T12:45:00Z"}
    // ... more users
  ]
}

POST /users

Create a new user account.

Request Body

{
  "username": "newuser",
  "email": "newuser@example.com",
  "password": "StrongP@ssw0rd!"
}

Response (201 Created)

{
  "id": 153,
  "username": "newuser",
  "email": "newuser@example.com",
  "created_at": "2024-11-02T09:12:45Z"
}

GET /users/{id}

Get details for a specific user.

Path Parameter

  • id (integer) – User ID.

Response (200 OK)

{
  "id": 101,
  "username": "alice",
  "email": "alice@example.com",
  "profile": {
    "first_name": "Alice",
    "last_name": "Smith",
    "avatar_url": "https://example.com/avatars/101.jpg"
  },
  "created_at": "2023-05-14T08:30:00Z",
  "last_login": "2024-10-30T14:22:10Z"
}

PUT /users/{id}

Update user information (requires ownership or admin).

Request Body (partial update allowed)

{
  "email": "alice.new@example.com",
  "profile": {"first_name": "Alicia"}
}

Response (200 OK)

{"status": "updated"}

DELETE /users/{id}

Delete a user account.

Response (204 No Content)

(No body)
